# Contributing

Thanks for your interest in contributing or using content from this repository. Below are some guidelines to make collaboration smooth and to ensure proper attribution.

## How to use
You’re welcome to reuse code and documentation from this repository for personal or commercial projects.

## Attribution requirement
If you reuse code or documentation, please include a short attribution in your README or source file header, for example:

- `Adapted from tioback/codility-lessons — https://github.com/tioback/codility-lessons`
- `Based on work by tioback (https://github.com/tioback/codility-lessons)`

If you prefer, add a short comment or header in the file noting the original source.

## Reporting & contributions
- Open an issue for bugs, missing citations, or if you find uncredited code.
- For code changes, open a pull request with a clear description and tests where applicable.
- Follow existing project style and add tests when relevant.

## Contact for attribution errors
If you believe your work is used here without proper credit, please open an issue or email me so I can correct it promptly.
